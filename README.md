# test_task

Программа протестирована на Windows 10 и Ubuntu.

Предполагается, что имена файлов для проверки содержат только стандартные символы - для того, чтоб использовать русский и прочие нестандартные
алфавиты, необходимо использовать кодировку файла со входными данными ANSI.

Была идея сделать многопоточную обработку следующим образом: создан пул потоков по числу ядер минус 1, в основном процессе читаются файлы
(операции доступа к жёсткому диску медленные, и разбиение их на потоки не выгодно), при прочтении каждый хэшируется в собственном потоке.
Это бы позволило выиграть по времени в случае наличия крупных файлов: в многопоточной архитектуре хэширование большого объёма данных не
блокировало бы выполнение хэширования множества мелких файлов. Я отказался от этой идеи, поскольку в таком случае для ускорения программы
пришлось бы загружать целые файлы в оперативную память перед хэшированием, и, в случае наличия крупных файлов, где подобное принесло бы
выигрыш по времени, оказалось бы занято слишком много памяти.
